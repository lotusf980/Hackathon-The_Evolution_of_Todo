# Feature Specification: Delete Task

**Feature Branch**: `2-delete-task`
**Created**: 2026-02-26
**Status**: Draft
**Input**: User provides ID → remove task if exists, otherwise clear error message

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Delete Existing Task (Priority: P1)

As a user, I want to delete a task by its ID, so that I can remove tasks that are no longer relevant.

**Why this priority**: Essential CRUD operation for task lifecycle management. Provides immediate value by allowing users to clean up their task list.

**Independent Test**: Can be fully tested by adding tasks, deleting one by ID, and verifying it is removed while others remain.

**Acceptance Scenarios**:

1. **Given** a task list with 3 tasks (IDs 1, 2, 3), **When** user deletes task with ID 2, **Then** task 2 is removed and tasks 1 and 3 remain.
2. **Given** a single task with ID 1, **When** user deletes task 1, **Then** the task list becomes empty.
3. **Given** multiple tasks, **When** user deletes a task, **Then** the system confirms deletion with a message like "Task deleted successfully."

---

### User Story 2 - Handle Invalid Delete Attempts (Priority: P2)

As a user, I want clear feedback when I try to delete a non-existent task, so that I understand the operation failed.

**Why this priority**: Prevents confusion when users mistype IDs or try to delete already-deleted tasks. Critical for user experience but secondary to core delete functionality.

**Independent Test**: Can be tested by attempting to delete invalid IDs and verifying appropriate error messages without crashes.

**Acceptance Scenarios**:

1. **Given** an empty task list, **When** user attempts to delete any ID, **Then** system displays "Error: No tasks exist."
2. **Given** tasks with IDs 1, 2, 3, **When** user attempts to delete ID 5, **Then** system displays "Error: Task with ID 5 not found."
3. **Given** a task list, **When** user enters a non-numeric ID, **Then** system displays "Error: Please enter a valid number."
4. **Given** a task list, **When** user enters a negative ID or zero, **Then** system displays "Error: Task ID must be a positive number."

---

### Edge Cases

- What happens when user deletes the last remaining task? → Task list becomes empty, operation succeeds
- How does system handle deleting a task that was already deleted? → Error message "Task not found"
- What happens to ID sequence after deletion? → IDs are NOT renumbered (remaining tasks keep original IDs)
- How does system handle rapid successive deletes? → Each delete processed sequentially

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST prompt user to enter the ID of the task to delete.
- **FR-002**: System MUST validate that the entered ID is a positive integer.
- **FR-003**: System MUST verify that a task with the specified ID exists before attempting deletion.
- **FR-004**: System MUST remove the task from the in-memory task list when ID is valid.
- **FR-005**: System MUST display a success confirmation message after successful deletion.
- **FR-006**: System MUST display "Error: No tasks exist." when user attempts to delete from an empty list.
- **FR-007**: System MUST display "Error: Task with ID {id} not found." when the ID does not match any task.
- **FR-008**: System MUST display "Error: Please enter a valid number." for non-numeric input.
- **FR-009**: System MUST display "Error: Task ID must be a positive number." for zero or negative integers.
- **FR-010**: System MUST NOT renumber remaining tasks after deletion (IDs remain unchanged).
- **FR-011**: System MUST return to the main menu after deletion or error.

### Key Entities

- **Task**: (See Add Task spec) - No changes to Task entity structure
- **TaskList**: In-memory collection of tasks with the following behavior:
  - Supports removal of tasks by ID
  - Maintains original IDs for remaining tasks after deletion
  - Returns count of tasks for empty-list validation

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Users can delete an existing task in under 10 seconds from viewing the task list.
- **SC-002**: 100% of delete operations on valid IDs successfully remove the correct task.
- **SC-003**: 100% of invalid delete attempts (non-existent ID, empty list, invalid format) produce appropriate error messages without crashes.
- **SC-004**: Remaining tasks retain their original IDs after deletion (no renumbering occurs).
